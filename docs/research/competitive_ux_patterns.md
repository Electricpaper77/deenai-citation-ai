# DeenAI Competitive UX Pattern Notes

Internal research summary for product positioning. This file records high-level public UX patterns only. It does not copy religious text, rulings, competitor wording, branding, icons, CSS, or source material.

## Positioning Constraint

DeenAI remains a citation-first Islamic AI reliability prototype using fixture-only validation artifacts. It does not claim scholar approval, issue fatwas or rulings, call a live LLM, run live RAG, or render live religious answers.

## Pattern Matrix

| Website | Strong UX Pattern | Strong Trust Pattern | Source/Citation Pattern | DeenAI Should Learn | DeenAI Should Avoid |
| --- | --- | --- | --- | --- | --- |

## Public-Site Distillation

The public site should use this research only as compact trust UX copy:

- Clear boundary: no fatwas, rulings, or live religious answers.
- Citation-gated rendering: answer surfaces require source IDs, citation requirements, and claim mapping.
- Reviewer proof path: fixture metrics, JSONL traces, and proof-chain summaries support fast review.
- Human escalation: personal or high-risk questions are escalated instead of answered.
